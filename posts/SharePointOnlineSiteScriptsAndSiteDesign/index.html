SharePoint Online Site Script

Mar 19, 2023 2 min readPowerShellSharePoint

What is Site Script

SharePoint Site Script is a JSON file used to define custom configurations within a SharePoint Online site. It can include a series of actions, such as creating lists, creating columns, setting site themes, tigger Flow etc., which can be automatically applied to a SharePoint site to ensure it adheres to specific standards and requirements. Site Scripts are reusable scripts that can be shared and reused across multiple SharePoint sites.

Using Site Scripts, administrators can apply them to existing sites or use them as templates when creating new sites. Site Scripts make it easy for administrators to customize SharePoint sites to meet organizational or business requirements. Site Scripts can also be used with the SharePoint Site Designer to help create custom site designs.

NOTE: Site Scripts can only be used in SharePoint Online and not in SharePoint Server.

The following example is a script that has 4 top-level actions. First, it applies a theme that was previously created named Contoso Explorers, sets regional settings, sets site logo and creats a “Customer Tracking” list.

{
  "$schema": "https://developer.microsoft.com/json-schemas/sp/site-design-script-actions.schema.json",
  "actions": [
    {
      "verb": "applyTheme",
      "themeName": "Contoso Explorers"
    },
    {
      "verb": "setRegionalSettings",
      "timeZone": 11,
      "sortOrder": 25,
      "hourFormat": "24"
    },
    {
      "verb": "setSiteLogo",
      "url": "/Style Library/logo.jpg"
    },
    {
      "verb": "createSPList",
      "listName": "Customer Tracking",
      "templateType": 100,
      "subactions": [
        {
          "verb": "setDescription",
          "description": "List of Customers and Orders"
        },
        {
          "verb": "addSPField",
          "fieldType": "Text",
          "displayName": "Customer Name",
          "isRequired": false,
          "addToDefaultView": true
        }
      ]
    }
  ],
  "version": 1
}

How to deploy a site script

Before you can use your site script, you need to deploy it to your tenant. You can manage your site scripts via PowerShell or REST.

Use PnP PowerShell to deploy a site script
  1. Connect to your SharePoint Online tenant using the Connect-PnPOnline cmdlet
  2. Upload the Site Script to tenant using Add-PnPSiteScript cmdlet
  3. Validate the uploaded site script using Get-PnPSiteScript cmdlet
Connect-PnPOnline -Url https://yourtenant.sharepoint.com/sites/yoursite -Interactive
$content=Get-Content "C:\Path\To\mysitescript.json" -Raw
Add-PnPSiteScript -Title "My Site Script" -Description "Custom site script" -Content $content -Version 1
Get-PnPSiteScript -Identity <SiteScriptID>
To update site script
Set-PnPSiteScript -Identity <SiteScriptID> -Title <Title> -Description <Description> -Content <Content>
To delete site script
Remove-PnPSiteScript -Identity <SiteScriptID>
